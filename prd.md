# PRD - dude-tax

- 产品显示名：工资薪金个税计算器

## 背景

面向财务或薪酬场景，提供单位维度的工资薪金个税管理工具。系统需要长期保存历年数据，同时允许快速试算与正式结果隔离。

## 核心用户

- 财务人员
- 薪酬专员
- 需要临时复核个税结果的内部使用者

## 核心价值

- 降低人工核算与核对成本
- 用“单位 + 年份”统一工作房间，降低误操作
- 用“月份确认”替代散乱的结果快照，提升追溯性与可维护性

## 当前优先级

### P0

- 工程可启动
- 单位管理
- 年份上下文
- 首页总览

### P1

- 员工信息
- 月度数据录入
- 缴纳确认
- 历史查询

### P2

- 快速计算
- 政策参考
- 系统维护

## 当前需求口径

- 月度数据录入以年度员工列表为主入口，不再以单员工单月为核心入口
- 只要员工在本年度任意时间在职，就出现在年度列表
- 月度录入和快速计算共享同一套工作台字段口径
- 录入入职前或离职后收入时，系统必须强提示并支持继续、跳过异常月份或取消
- 缴纳确认必须按月份顺序推进，并且只允许确认覆盖了全部有效员工的当前结果
- 历史查询只看已确认数据
- 系统维护提供单位备份导出入口

## 数据库存储约束

- 开发态 `npm run dev` 必须使用独立开发数据库：`data/dev/dude-tax.dev.db`
- 已安装 Windows 软件默认使用安装目录下的 `data/dude-tax.db`
- 已安装软件不得再与开发环境共用同一 SQLite 数据库
- 用户升级到新版本时，如旧 `userData` 库存在且安装目录下尚无新库，系统应自动迁移旧数据到安装目录路径

## 成功标准

- 软件可以在本机启动
- 可创建单位并切换年份
- 可维护员工信息
- 可完成员工与月度数据批量导入
- 可在月度数据录入页执行年度计算并查看结果
- 可顺序确认月份并锁定已确认月份
- 可按单位、年份查询历史结果
- 可执行不落库的快速试算
- 可为当前单位导出 ZIP 备份
- 开发态与安装版数据库相互隔离，不再串库
