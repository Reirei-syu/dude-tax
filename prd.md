# PRD - dude-tax

- 产品显示名：工资薪金个税计算器

## 背景

面向财务或薪酬使用场景，提供单位维度的工资薪金个税管理工具。系统需要长期保存历年数据，同时允许快速测算与正式结果分离。

## 核心用户

- 财务人员
- 薪酬专员
- 需要临时复核个税结果的内部使用者

## 核心价值

- 降低人工核算与核对成本
- 将“单位 + 年份”作为统一工作房间，减少误操作
- 用“月份确认”替代散乱的结果快照，提升可追溯性和可维护性
- 让“已录入内容就是计算来源”，减少状态型字段带来的理解成本

## 当前优先级

### P0

- 工程可启动
- 单位管理
- 年份上下文
- 首页总览

### P1

- 员工信息
- 月度数据录入
- 缴纳确认
- 历史查询

### P2

- 快速计算
- 政策参考
- 系统维护

## 当前需求口径

- 月度数据录入以年度员工列表为主入口，而不是单员工单月入口
- 只要员工在本年度任意时间在职，就出现在年度列表
- 月度数据录入页默认纳入全部有效员工，并支持通过“选择员工”弹层调整当前名单
- 选择员工弹层必须区分“在职员工”和“本年离职员工”，以前年度离职员工不展示
- 编辑窗口采用应用内可最大化工作台，不开独立 Electron 子窗口
- 月度数据录入页提供与快速计算同口径的“预扣模式 + 执行计算”入口
- 员工批量导入并入员工信息模块，保留模板下载、导入预览、冲突处理和执行导入
- 月度数据批量导入并入月度数据录入模块，保留模板下载、导入预览、冲突处理和执行导入
- 月度模板覆盖当前单位本年度全部有效员工，每名员工固定 12 行
- 用户删除月度模板中的员工/月份行时，对应月份按 0 数据导入
- 月度数据录入页计算完成后，直接展示结果汇总，并可查看单员工全年计算结果明细
- 主卡片“月度数据手工录入”统一承载员工列表编辑、计算控制与结果覆盖摘要
- 计算结果列表支持折叠，避免长名单影响使用体验
- 若员工在入职前月份或离职后月份录入了收入，系统必须强提示，并允许继续、仅跳过异常月份或取消
- 左侧主导航支持一键抽屉式收起与恢复，释放更多工作区面积
- 左侧导航按钮支持上下拖拽换位，并在重启后保留顺序
- 顶部“单位 + 年份”状态卡保持固定，不允许拖拽、缩放或重排
- 工作区主卡片支持从非交互区域拖拽移动、右下角描边提示缩放、自动对齐和页面级布局记忆
- 卡片与主要工作台弹窗的正文区在缩放时需要整体等比缩放内容
- 主要工作台弹窗支持拖拽、缩放、最大化 / 还原，并在重启后保留窗口布局
- 快速计算采用单案例模式，不写正式数据
- 缴纳确认必须按月份顺序推进，并允许级联取消确认
- 缴纳确认展示当前待确认结果，并且只有在当前结果覆盖全部有效员工时才允许确认月份
- 缴纳确认主工作卡片名称为“已纳税月份确认”，月份按钮颜色直接表示确认状态
- 历史查询只看已确认数据
- 补发收入统一为“其他收入”，不再使用补扣税调整
- 工作台不再维护“记录状态”
- 减除费用按当前政策默认值只读预填，默认 5000
- Excel 导出统一采用“汇总 sheet + 每员工 1 个 sheet”
- 月度录入旧导出按钮取消，导出入口改放到“计算结果汇总”卡片
- 系统维护提供“单位备份”入口，按当前单位导出全部年份业务数据 ZIP
- 单位备份需记住最近一次成功备份目录，便于后续直接复用
- 本期只做备份，不做恢复

## 成功标准

- 软件可以在本机启动
- 可创建单位并切换年份
- 可维护员工信息
- 可在员工信息页直接完成员工模板下载、预览和批量导入
- 可在月度数据录入页快速查看年度员工全局情况
- 可在工作台内维护 12 个月数据并批量复制到下月 / 后续月份
- 当录入或复制收入到入职前 / 离职后月份时，会出现强提示且后端也会阻断未确认写入
- 可在月度数据录入页直接完成月度模板下载、预览和批量导入
- 可在月度数据录入页直接执行年度计算，并查看员工汇总与单员工结果明细
- 可对月份数据执行顺序确认，并在结果覆盖全部有效员工后锁定已确认月份
- 可在空白月直接执行确认，并在后续结果中按零值月参与计算
- 可按单位、年份查询已确认历史数据
- 可从计算结果汇总卡片导出当前结果 Excel，格式为“汇总 + 每员工 1 个 sheet”
- 可通过快速计算进行不落库试算
- 可在系统维护页为当前单位生成 ZIP 备份，并在下一次备份时默认带出最近目录
