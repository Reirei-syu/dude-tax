# PRD - dude-tax

- 产品显示名：工资薪金个税计算器

## 背景

面向财务或薪酬使用场景，提供单位维度的工资薪金个税管理工具。系统需要长期保存历年数据，同时允许快速测算与正式结果分离。

## 核心用户

- 财务人员
- 薪酬专员
- 需要临时复核个税结果的内部使用者

## 核心价值

- 降低人工核算与核对成本
- 将“单位 + 年份”作为统一工作房间，减少误操作
- 用“月份确认”替代散乱的结果快照，提升可追溯性和可维护性
- 让“已录入内容就是计算来源”，减少状态型字段带来的理解成本

## 当前优先级

### P0

- 工程可启动
- 单位管理
- 年份上下文
- 首页总览

### P1

- 员工信息
- 月度数据录入
- 结果确认
- 历史查询

### P2

- 快速计算
- 批量导入
- 政策参考
- 系统维护

## 当前需求口径

- 月度数据录入以年度员工列表为主入口，而不是单员工单月入口
- 只要员工在本年度任意时间在职，就出现在年度列表
- 编辑窗口采用应用内可最大化工作台，不开独立 Electron 子窗口
- 快速计算采用单案例模式，不写正式数据
- 结果确认必须按月份顺序推进，并允许级联取消确认
- 历史查询只看已确认数据
- 补发收入统一为“其他收入”，不再使用补扣税调整
- 工作台不再维护“记录状态”
- 减除费用按当前政策默认值只读预填，默认 5000
- Excel 导出统一采用“汇总 sheet + 每员工 1 个 sheet”

## 成功标准

- 软件可以在本机启动
- 可创建单位并切换年份
- 可维护员工信息
- 可在月度数据录入页快速查看年度员工全局情况
- 可在工作台内维护 12 个月数据并批量复制到下月 / 后续月份
- 可对月份数据执行顺序确认，并锁定已确认月份
- 可在空白月直接执行确认，并在后续结果中按零值月参与计算
- 可按单位、年份查询已确认历史数据
- 可导出月度录入、结果确认、历史查询三类 Excel
- 可通过快速计算进行不落库试算
