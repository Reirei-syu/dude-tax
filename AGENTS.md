# 项目级执行规则

## 项目定位
- 项目标识：dude-tax
- 产品显示名：工资薪金个税计算器
- 当前阶段：v0.1.0-alpha
- 目标形态：Windows 单机离线桌面软件
- 架构方向：Electron + React + 本地 API + SQLite

## 当前开发原则
- 采用小步快跑模式，每次只完成一个可运行、可验证、可回退的里程碑
- 严禁在单次迭代中同时展开多个核心模块
- 每完成一步都必须先测试、再 Review，确认无明显问题后进入下一步
- 优先保留上下文记忆，所有关键决策必须同步写入 `PROGRESS.md`
- 每次任务完成后，输出中必须明确给出“下一步建议”
- 当项目达到“初步可用闭环”时，必须明确告知“项目已初步完成”，并补充后续优化建议

## 当前里程碑范围
- 初始化项目文档
- 搭建前后台工程骨架
- 建立数据库初始化能力
- 实现“单位 + 年份”全局上下文闭环
- 提供首页与单位管理的最小可运行界面
- 提供员工信息模块最小闭环
- 提供月度数据录入模块最小闭环
- 提供计算中心最小闭环
- 提供历史查询模块最小闭环
- 提供历史查询重算版本历史最小闭环
- 提供系统维护模块的税率维护与版本切换最小闭环
- 提供系统维护模块的审计日志、作用域解绑恢复继承与版本影响预览最小闭环
- 提供快速计算模块最小闭环
- 提供批量导入模块最小闭环
- 提供 Windows 测试包脚本与桌面自启本地 API 的最小交付链路

## 当前明确不做
- 个税计算引擎详细实现
- 历史结果版本对比
- 备份恢复

## 分层约束
- `apps/desktop`：桌面壳与前端 UI
- `apps/api`：本地 API 服务
- `packages/core`：核心领域模型、校验与后续计算逻辑
- `packages/config`：默认配置与静态常量

## 关键业务约定
- “单位名称”是用户可见概念，系统内部按独立数据空间处理
- 用户必须先选定“单位 + 年份”，再进入后续工作模块
- 税率标准和提示说明为全局级，不按单位隔离
- 当前税率支持历史版本、全局活动版本以及单位 / 年度作用域绑定；结果与重算记录按当前作用域税率逻辑判定失效或恢复有效
- 桌面测试包默认将数据库写入用户数据目录，不再依赖仓库 `data/` 目录
- 桌面测试包通过编译后的本地 API 产物运行，不允许在生产环境依赖 `tsx` 或 `apps/api/src/*.ts`
- 默认自动采用税额较低方案；用户可手动切换到另一方案
- 月度录入阶段先只做数据维护，不做税额试算或结果输出
- 计算中心当前只做准备状态与重算时间占位，不输出税额结果
