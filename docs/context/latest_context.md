# 当前上下文摘要

## 当前阶段
- 项目处于 `v0.1.0-alpha`，主业务闭环已覆盖单位管理、员工信息、月度录入、年度计算、结果查看与 CSV / XLSX 导出。
- 当前正在做结果导出体验打磨，优先补齐财务实际使用时最依赖的 Excel 可读性能力。

## 已完成模块
- Electron + React + Fastify + SQLite 本地桌面骨架可运行。
- “单位 + 年份” 全局上下文闭环已接通。
- 员工信息、月度录入、年度计算、结果中心主链路已跑通。
- 结果中心支持手动切换计税方案、导出字段选择、导出模板、CSV / XLSX 双格式导出。
- XLSX 导出已补充分组表头配色、自动列宽、冻结首行、自动筛选与补退税颜色提示。

## 当前任务
- 本轮已完成 XLSX 导出格式增强，并补充对应测试。
- 下一步优先处理桌面端文件保存对话框、多工作表说明页和更完整的文件落地体验。

## 关键设计决策
- 继续遵守最小改动原则，仅增强 `apps/desktop/src/pages/annual-tax-export.ts` 与结果页文案，不调整现有导出数据结构。
- 导出字段顺序仍由既有模板和列定义统一驱动，CSV / XLSX 共用同一套字段选择逻辑。
- Excel 样式只做轻量增强，避免引入额外导出状态或侵入式 UI 配置。

## 未解决问题
- Vite 构建在当前沙箱环境下会触发 `spawn EPERM`，因此本轮未拿到新鲜构建通过证据。
- XLSX 仍缺少多工作表、保存对话框和更细的财务模板样式。
- `exceljs` 引入后桌面端体积偏大，后续要评估按需加载或代码拆分。

## 下一步计划
- 接入 Electron 保存对话框或等价桌面落地能力，减少浏览器式下载体验。
- 为 XLSX 增加说明页或多工作表结构，提升对外交付可读性。
- 继续推进历史查询与税标维护模块。
