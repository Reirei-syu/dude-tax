# 当前上下文摘要

## 当前阶段
- 项目处于 `v0.1.0-alpha`，主业务闭环已覆盖单位管理、员工信息、月度录入、年度计算、结果查看、历史查询最小闭环，以及 CSV / XLSX 双格式导出。
- 当前代码已通过 `npm run typecheck` 与 `npm test` 验证，本轮继续补齐了系统维护模块的只读入口，并将仓库整理为只保留 `master`。

## 已完成模块
- Electron + React + Fastify + SQLite 本地桌面骨架可运行。
- “单位 + 年份” 全局上下文闭环已接通。
- 员工信息、月度录入、年度计算、结果中心主链路已跑通。
- 结果中心支持手动切换计税方案、导出字段选择、导出模板、CSV / XLSX 双格式导出。
- 历史查询支持按单位、年份、员工和结算方向筛选，并提供只读详情。
- 系统维护已提供只读页，可查看默认税标、维护边界和后续开放计划。
- `PENDING_GOALS.md` 已沉淀未完成目标清单，便于后续迭代拆解。

## 当前任务
- 本轮已完成系统维护只读页落地，并移除剩余本地备份分支。
- 当前主分支已经收口，后续可继续从系统维护可编辑版本或导出体验优化继续推进。

## 关键设计决策
- 继续遵守最小改动原则，不调整既有分层边界，仍保持 `UI -> Service/API -> Core` 的调用方向。
- 历史查询阶段先读取最新年度结果快照，不引入历史版本重算链路，避免本轮扩大范围。
- 系统维护本轮只落地只读页，直接复用 `packages/config` 静态税标，不提前引入持久化表与 API 改造。
- 仓库整理采用“临时分支提交 -> 合并回 master”的方式，而不是直接在脏工作区提交，优先保证可回退性。

## 未解决问题
- 当前仓库没有配置远程仓库，后续若要协作或备份，需要补充 `origin` 与分支协作规范。
- 系统维护仍不支持税标编辑、持久化、版本管理和结果失效联动。
- 历史查询仍不包含重算历史版本、历史导出和差异对比能力。
- XLSX 导出仍缺少更完整的财务样式、多工作表与桌面文件落地体验。
- `exceljs` 引入后桌面端 bundle 体积偏大，后续要评估按需加载或代码拆分。

## 下一步计划
- 继续推进系统维护模块，补齐税标编辑、全局配置持久化和结果失效联动。
- 继续优化 Excel 导出体验，补充样式模板、列宽、冻结表头和文件保存体验。
- 后续再补齐历史查询增强、批量导入，以及更复杂的预扣预缴与跨年口径。
