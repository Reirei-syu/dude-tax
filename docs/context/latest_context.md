# 当前上下文摘要

## 当前阶段
- 项目处于 `v0.1.0-alpha`
- 项目已初步完成，主要业务模块、前后台数据勾连和模块联动均已跑通
- 当前重点转向导出模板策略、复杂税务口径、数据围栏和边界收口
- 当前代码已通过 `npm run typecheck` 与 `npm test`

## 已完成模块
- Electron + React + Fastify + SQLite 本地桌面骨架可运行
- “单位 + 年份”全局上下文闭环已接通
- 员工信息、月度录入、计算中心、结果中心、历史查询、系统维护、快速计算、批量导入均已具备最小可运行闭环
- 系统维护已支持税率编辑、富文本说明维护、版本激活、单位 / 年度作用域绑定和逻辑失效判定
- 首页已联动待重算、未完成月份、导入冲突和工作建议
- 历史查询已支持结果筛选、状态切换、跨年度概览、CSV / XLSX 导出、失效快照差异对比和重算版本历史
- 结果中心已支持历史版本查看和版本差异对比
- 批量导入已支持 CSV、XLSX、XLSM 文件读取，以及细粒度预览展示

## 当前任务
- 本轮已完成 `系统维护 -> 富文本说明维护`
- 已在系统维护页补齐富文本说明编辑、格式工具栏和预览闭环

## 关键设计决策
- 富文本说明继续使用现有说明字符串字段保存，不新增数据库字段和后端接口
- 富文本能力采用轻量标记语法 + 前端安全渲染，不引入额外富文本编辑器依赖
- 继续保持 `UI -> Service/API -> Core` 分层，不在前端直接拼后端业务规则

## 未解决问题
- 结果中心仍缺少导出结果模板管理
- 个税计算核心仍未覆盖完整预扣预缴、补发补扣、跨单位 / 跨年衔接等复杂口径
- `exceljs` 引入后桌面端 bundle 体积偏大，后续仍需评估按需加载或拆分
- 当前仓库无远程仓库，后续如需协作仍需补远程与分支规范

## 下一步
- 优先推进 `结果中心 -> 导出结果模板管理`
- 后续进入更复杂税务口径、导出模板策略和数据围栏优化
