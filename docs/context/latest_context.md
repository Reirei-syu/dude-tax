# 当前上下文摘要

## 当前阶段
- 项目处于 `v0.1.0-alpha`
- 主要业务模块已基本搭齐，当前重点转向模块联动优化、边界收口和数据围栏
- 当前代码已通过 `npm run typecheck` 与 `npm test`

## 已完成模块
- Electron + React + Fastify + SQLite 本地桌面骨架可运行
- “单位 + 年份”全局上下文闭环已接通
- 员工信息、月度录入、计算中心、结果中心、历史查询、系统维护、快速计算、批量导入均已具备最小可运行闭环
- 系统维护已支持税率编辑、版本激活、单位 / 年度作用域绑定和逻辑失效判定
- 首页已联动待重算、未完成月份、导入冲突和工作建议
- 历史查询已支持结果筛选、状态切换、跨年度概览、CSV / XLSX 导出、失效快照差异对比
- 历史查询已支持查看所选员工年度结果的重算版本历史
- 结果中心已支持查看当前员工年度结果的历史版本时间线
- 结果中心已支持对两个历史版本进行关键字段差异对比

## 当前任务
- 本轮已完成 `结果中心增强 -> 结果版本差异对比`
- 已在结果中心支持选择两个历史版本，并展示关键字段差异

## 关键设计决策
- 保留 `annual_tax_results` 作为当前结果投影表，不把历史版本直接塞进当前结果表
- 新增 `annual_tax_result_versions` 作为追加式快照表，仅记录真实重算产生的版本
- 手动方案切换仍只更新当前投影，不额外写入“重算版本历史”
- 继续保持 `UI -> Service/API -> Core` 分层，不在前端直接拼业务规则

## 未解决问题
- 结果中心仍缺少导出结果模板管理
- 批量导入仍不支持 Excel 解析
- 批量导入预览仍缺少更细粒度字段展示
- 系统维护仍未支持富文本说明维护
- 个税计算核心仍未覆盖完整预扣预缴、补发补扣、跨单位 / 跨年衔接等复杂口径
- `exceljs` 引入后桌面端 bundle 体积偏大，后续仍需评估按需加载或拆分

## 下一步
- 优先推进 `结果中心增强 -> 导出结果模板管理`
- 后续再推进 `批量导入 -> Excel 解析支持`
- 批量导入的 Excel 解析和预览细化继续放在结果版本化能力之后
