# 当前上下文摘要

## 当前阶段
- 项目处于 `v0.1.0-alpha`
- 当前目标是收口“真实业务试点上线边界”
- 主链功能、测试、类型检查、桌面前端构建均已通过

## 本轮已完成
- 桌面端改为运行时注入 API 地址，不再把 `127.0.0.1:3001` 写死在前端
- 桌面壳已加入本地 API 子进程启动逻辑，并把默认数据库目录迁移到用户数据目录
- 系统维护补齐税率变更审计日志、作用域解绑恢复继承、版本差异与影响预览
- 结果中心与历史查询补齐“规则来源 / 外部衔接说明”
- 历史查询失效快照对比改为按当前作用域税率重算
- `apps/*` 对 `packages/*/src` 的深相对路径已收口到 workspace 包名
- `npm audit --omit=dev` 已为 0 漏洞
- 已新增 `npm run package:win`，可生成 `dist-electron/` Windows 测试包目录

## 当前阻塞
- Windows 测试包 smoke 仍未通过
- 打包后 `better-sqlite3` 仍存在 Electron ABI 不匹配，导致本地 API 子进程未成功启动
- 因此当前测试包尚不能完成“启动即初始化数据库 + 本地 API 健康检查通过”的验收

## 关键决策
- 试点包当前采用 `@electron/packager` 打包，先以可分发目录为目标，不先做安装器
- 试点包当前关闭 `asar`，优先保证桌面壳对子进程脚本和运行时资源的直接访问
- 系统维护的版本影响预览默认以“当前单位 / 年度作用域”作为比较范围
- 当前不把更多特殊税务场景纳入本轮，先把现有试点边界收口

## 下一步
- 优先处理 `better-sqlite3` 的 Electron ABI 重编，使测试包 smoke 通过
- 随后继续补真实业务试点范围内的复杂税务口径与显式警示
