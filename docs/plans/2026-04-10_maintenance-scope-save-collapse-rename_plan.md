# 系统维护模块折叠、作用域保存修复与版本命名方案

## 1. 背景
- 系统维护页当前读取的是作用域税率版本，但保存时仍按全局税率版本落库。
- 这会导致当前作用域已绑定历史版本时，新增或编辑扣除项说明后，政策参考页看不到刚保存的内容。
- 同时，系统维护页卡片过长、税率表在非全屏下容易溢出，税率版本也缺少名称维护能力。

## 2. 目标
- 修复系统维护页在当前作用域下保存政策说明与税率配置的落库行为。
- 所有系统维护卡片支持默认折叠。
- 扣除项说明和基本减除费用改为上下结构。
- 税率表输入区域在非全屏下自适应卡片宽度。
- 税率版本列表支持行内编辑名称，并以用户本地时间显示创建时间与启用时间。

## 3. 设计方案
- 保存接口新增 `unitId`、`taxYear`，让仓储根据当前作用域决定更新当前绑定版本、重绑版本或激活全局版本。
- 版本改名新增独立 `PATCH /api/tax-policy/versions/:versionId` 路由，只更新名称与审计日志。
- 系统维护页为概览、扣除项说明、基本减除费用、综合所得税率表、年终奖税率表、税率版本列表、版本影响预览、审计日志八类卡片统一接入折叠状态，默认全折叠。
- 扣除项说明与基本减除费用卡片改为全宽卡片；税率表保留原布局，仅增加自适应包裹容器并收紧输入框宽度。

## 4. 涉及模块
- `packages/core`
- `apps/api`
- `apps/desktop`
- `docs`

## 5. 数据结构变更
- `TaxPolicyUpdatePayload` 新增 `unitId`、`taxYear`、`versionName`
- `TaxPolicyAuditAction` 新增 `rename_version`
- 新增 `TaxPolicyVersionRenamePayload`

## 6. 接口变更
- `PUT /api/tax-policy` 支持作用域保存
- `PATCH /api/tax-policy/versions/:versionId` 支持重命名税率版本

## 7. 风险评估
- 若作用域上下文传递不完整，会继续误改全局版本。
- 若版本重命名未隔离为独立接口，容易与税率保存逻辑混淆。
- 若税率表仍保留输入框最小宽度，页面在窄窗口下仍可能溢出。

## 8. 回退方案
- 可回退到全局保存逻辑，但需同时撤销作用域保存测试。
- 可禁用版本改名接口，保留自定义命名仅用于新建版本。
- UI 折叠与布局可单独回退，不影响税率计算核心逻辑。

## 9. 任务拆解
- 扩展 core 类型与 API 客户端。
- 修复税率仓储保存逻辑并新增版本改名接口。
- 更新系统维护页折叠、布局、自定义版本名与行内改名交互。
- 补充 API / Desktop 自动化测试并同步文档。
