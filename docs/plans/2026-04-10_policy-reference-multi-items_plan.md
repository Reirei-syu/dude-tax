# 系统维护与政策参考多条说明条目方案

## 1. 背景
- 当前系统维护模块中的“扣除项说明”只支持维护单条 `标题 + 正文 + 插图`。
- 政策参考模块也只渲染同一条说明，无法同时展示多条政策说明。
- 用户已明确要求支持添加若干条说明，并对已添加条目持续维护。

## 2. 目标
- 将单条政策说明升级为多条可维护条目。
- 系统维护模块支持新增与编辑多个政策说明条目。
- 政策参考模块按条目列表展示这些说明。
- 保持旧数据兼容，不新增数据库表，不破坏现有税率版本与审计逻辑。

## 3. 设计方案
### 3.1 存储策略
- 继续复用 `tax_policy_versions.maintenance_notes` 字段存储说明内容。
- 将该字段的 JSON 结构从单条对象升级为 `policyItems` 数组。
- 兼容旧数据：
  - 旧 JSON 单条对象自动迁移为数组中的第一条。
  - 纯文本旧说明自动迁移为仅有正文的一条说明。

### 3.2 Core 类型调整
- 新增 `TaxPolicyItemInput` 与 `TaxPolicyItem` 类型。
- `TaxPolicyResponse` 新增：
  - `policyItems`
  - `defaultPolicyItems`
- 旧单条字段 `policyTitle / policyBody / policyIllustrationDataUrl` 保留，作为第一条说明的兼容别名。

### 3.3 API 层调整
- `PUT /api/tax-policy` 支持接收 `policyItems`。
- 仓储层统一负责：
  - 兼容解析旧结构
  - 过滤完全空白的条目
  - 规范化 `id` 与 `sortOrder`
  - 将第一条条目回写到旧兼容字段

### 3.4 前端系统维护页
- “扣除项说明”模块增加“新增说明条目”按钮。
- 每条说明以独立编辑卡片展示，支持：
  - 标题编辑
  - 正文编辑
  - 插图上传与清空
  - 富文本快捷按钮
  - 实时预览
- 本轮只做新增与编辑，不做删除与拖拽排序。

### 3.5 政策参考页
- 将单条说明展示改为多条说明列表展示。
- 每条说明独立显示标题、插图与正文。
- 无条目时继续显示空状态提示。

## 4. 涉及模块
- `packages/core`
- `apps/api`
- `apps/desktop`
- `docs`

## 5. 数据结构变更
- 不新增数据库表。
- `maintenance_notes` 序列化内容新增 `policyItems` 数组。
- 旧结构读取兼容，新结构写入生效。

## 6. 接口变更
- `GET /api/tax-policy` 响应新增 `policyItems`、`defaultPolicyItems`。
- `PUT /api/tax-policy` 请求支持 `policyItems`。
- 旧单条字段继续兼容，不立即移除。

## 7. 风险评估
- 风险 1：旧说明内容解析兼容失败会导致政策参考页为空。
  - 处理：增加旧结构与纯文本兼容测试。
- 风险 2：系统维护页改成多条编辑后，验证逻辑可能遗漏单条超长文本问题。
  - 处理：将前端校验从单条正文扩展到逐条校验。
- 风险 3：政策参考页和系统维护页若一端仍读旧字段，会出现显示不一致。
  - 处理：前后端统一切到 `policyItems`，旧字段仅保留兼容别名。

## 8. 回退方案
- 若多条条目实现异常，可回退为只读取第一条条目。
- 若新数组结构解析异常，可继续回退读取旧单条 `policyTitle / policyBody / policyIllustrationDataUrl`。
- 因未改数据库 schema，回退只需恢复代码逻辑，不需要迁移脚本。

- 更新桌面端测试、任务文档、进度文档和上下文快照。
