# 不变量与收敛条件

## 1. 全局不变量

- [ ] 停止导入与重算后，`annual_tax_result_versions` 不再持续增长。
- [ ] 同一数据、同一税率、同一选择方案的重复重算不会新增版本历史。
- [ ] `annual_tax_results`、`annual_calculation_runs`、`month_confirmations` 与 `employee_month_records` 可相互对账。
- [ ] 数据库文件与 WAL 在无变更重算阶段增长率不超过基线后的 10%。
- [ ] API 重启后，查询链路恢复正常，不出现结果丢失、版本放大或确认状态漂移。

## 2. 关键链路专用不变量

### 链路 A：无变更重复重算

- 触发：对同一单位 / 年度重复调用重算接口
- 期望稳定态：`annual_tax_result_versions` 总数保持与首次有效结果条数一致
- 失败阈值：任一员工在无变更场景下出现第 2 条新增版本

### 链路 B：已确认结果列表查询

- 触发：500 员工、12 个确认月份下执行列表与明细查询
- 期望稳定态：基线请求 ≤ 1.5 秒，收尾请求 ≤ 2.0 秒
- 失败阈值：基线或收尾延迟超过阈值

### 链路 C：恢复阶段

- 触发：API 重启后重新执行重算、已确认结果、历史查询
- 期望稳定态：结果总数、版本总数与重启前对齐
- 失败阈值：重启后版本数增加、结果数减少或查询持续失败
