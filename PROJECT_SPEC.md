# dude-tax - 项目规格

- 产品显示名：工资薪金个税计算器
- 当前版本：v0.1.0-alpha
- 目标形态：Windows 单机离线桌面软件
- 架构方向：Electron + React + 本地 API + SQLite

## 1. 项目目标

- 提供单位级、年度级的工资薪金个税数据录入、确认、查询与导出能力
- 保持全离线运行，核心数据在本地可长期保存
- 支持不落库的快速试算，与正式业务数据隔离

## 2. 分层与模块

- `apps/desktop`：桌面壳与前端 UI
- `apps/api`：本地 API 与 SQLite 持久化
- `packages/core`：共享领域模型、校验、布局协议与计算相关纯逻辑
- `packages/config`：默认配置与静态常量

当前主要业务模块：

- 首页
- 单位管理
- 员工信息
- 月度数据录入
- 缴纳确认
- 历史查询
- 快速计算
- 政策参考
- 系统维护

## 3. 关键业务约束

- 全局工作上下文为“单位 + 年份”
- 税率标准与政策说明为全局能力，但支持历史版本、活动版本与单位/年度作用域绑定
- 月度录入阶段先做数据维护，不直接输出正式税额结果
- 计算中心当前仅提供准备状态与重算时间占位
- 历史查询只读取已确认数据
- 单位备份当前仅做导出，不做恢复

## 4. UI 与交互约束

- 顶部“单位 + 年份”状态卡固定，不参与自由布局
- 页面级主卡片支持拖拽、缩放、重叠摆放和显式“自动排列”
- 拖拽或缩放结束后不自动吸附、不自动避让
- 右键卡片非交互区支持“顶置 / 靠左 / 靠右”
- 可折叠卡片按页面 `scope` 记忆最后一次展开状态
- 主工作台弹窗支持拖拽、缩放、最大化/还原，并记忆布局

## 5. 数据与接口约束

- HTTP API 与前端业务接口在本轮数据库隔离改造中不变
- `runtimeConfig = { apiBaseUrl, managedApi, databasePath }`
- UI 偏好统一落到 SQLite `app_preferences`
- 年度结果快照必须同时匹配当前税率签名与数据签名，才视为可确认

## 6. 数据库存储策略

- 开发态 `npm run dev` 默认使用仓库内独立数据库：`data/dev/dude-tax.dev.db`
- 安装版 managed API 优先使用 `{安装目录}\data\dude-tax.db`
- 若安装目录不可写，允许自动回退到 `app.getPath("userData")\data\dude-tax.db`
- 升级到新策略时，若安装目录下尚无数据库但旧 `userData` 库存在，启动时自动复制迁移旧库，并同步复制 `-wal` / `-shm`
- 开发态与安装版数据库必须长期隔离，禁止再次共享同一 SQLite 文件

## 7. 非功能要求

- 全离线运行
- Windows 优先
- 所有关键链路可验证、可回退
- 桌面测试包通过编译后的本地 API 产物运行，不依赖 `tsx` 或 `apps/api/src/*.ts`
- 发布前脚本必须能验证安装版数据库实际落库路径
