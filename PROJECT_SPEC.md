# dude-tax - 项目规格

- 产品显示名：工资薪金个税计算器

## 1. 产品目标

- 提供单位级、年度级的工资薪金个税录入、确认、查询与导出能力
- 采用桌面端本地存储方式，支持历年数据长期保存
- 提供快速计算模块，用于不落库的临时试算

## 2. 顶层业务模型

- 全局工作上下文：`单位 + 年份`
- 全局政策上下文：`税率标准 + 政策参考说明`
- 正式业务主线：
  - 员工信息维护
  - 月度数据录入
  - 缴纳确认
  - 历史查询
- 临时业务主线：
  - 快速计算
  - 仅用于试算，不写入正式月度记录

## 3. 当前模块清单

- 首页
- 单位管理
- 员工信息
- 快速计算
- 月度数据录入
- 缴纳确认
- 历史查询
- 政策参考
- 系统维护

## 4. 当前功能边界

- 员工信息
  - 支持手工新增 / 编辑 / 删除员工
  - 内嵌“员工批量导入”区块，保留模板下载、导入预览、冲突策略和执行导入
- 月度数据录入
  - 主卡片“月度数据手工录入”统一展示年度员工列表编辑、计算控制与结果覆盖摘要
  - 默认纳入本年全部有效员工，支持通过“选择员工”弹层调整当前名单
  - 内嵌“月度数据批量导入”区块，保留模板下载、导入预览、冲突策略和执行导入
  - 选择员工弹层按“在职员工 / 本年离职员工”分组展示，以前年度离职员工不进入名单
  - 支持进入可最大化年度工作台编辑 12 个月数据
  - 工作台收入区使用“工资收入、年终奖、预扣税额、其他收入、其他收入备注”
  - 当在入职前月份或离职后月份录入收入时，需弹出强提示，并支持继续、跳过异常月份或取消
  - 工作台不再显示“记录状态”
  - 工作台展示只读“减除费用”列，值来自当前政策 `basicDeductionAmount`
  - 支持将当前月应用到下月 / 后续月份
  - 提供与快速计算同口径的“预扣模式 + 执行计算”入口
  - 计算结果汇总页列示“年度累计应预扣额、最后一个月适用税率、采用方案、另一方案应扣税额”
  - 计算结果汇总列表支持折叠
  - 明细弹层展示单员工全年计算结果，列示方式与快速计算模块一致
  - 主卡片不再提供旧“导出 Excel”按钮
  - 计算结果汇总卡片提供“导出当前结果”按钮，导出“汇总 sheet + 每员工 sheet”Excel
- 快速计算
  - 采用单案例总览 + 年度工作台
  - 不填写姓名等基础信息
  - 与月度录入使用同一套工作台字段
  - 支持将当前月应用到下月 / 后续月份
  - 支持基于当前单位 / 年份政策即时试算
- 缴纳确认
  - 合并原计算中心与结果中心
  - 模块导航名称为“缴纳确认”，主工作卡片名称为“已纳税月份确认”
  - 主视图展示当前待确认结果快照，而不是只看已确认结果
  - 按月份顺序确认，且仅当当前结果已覆盖全部有效员工时允许确认
  - 月份按钮颜色直接表达是否已确认，不再额外展示状态摘要卡
  - 确认后锁定对应月份数据
  - 取消确认时级联解除当前月及后续月份
  - 允许空白月直接确认
  - 已确认但无录入内容的月份按零值月参与计算
  - 明细弹层展示当前年度计算结果
  - 支持导出当前待确认结果汇总 Excel
- 历史查询
  - 只查询已确认数据
  - 条件保留单位、年份
  - 展示年度员工纳税列表与全年明细
  - 支持导出“汇总 sheet + 每员工 sheet”Excel
- 桌面交互
  - 左侧主导航支持抽屉式收起与恢复，以释放工作区宽度
  - 左侧导航按钮支持纵向拖拽排序，并记忆全局顺序
  - 顶部“单位 + 年份”状态卡固定，不参与工作区自由布局
  - 工作区主卡片支持从非交互区拖拽移动、右下角描边手柄缩放、自动对齐与页面级布局记忆
  - 卡片与主要弹窗正文区在缩放时按比例整体缩放内容
  - 主要工作台弹窗支持浮动窗口拖拽、缩放、最大化/还原，并记忆窗口位置与尺寸
- 系统维护
  - 支持按当前单位导出全部年份业务数据 ZIP 备份
  - 备份文件名默认采用“单位名称 + 日期时间”
  - 支持记住最近一次成功备份目录，用于下次直接备份或重新选路径

## 5. 当前数据与接口约束

- 数据库保留旧 `status`、补扣税调整、所属期间字段用于兼容，不做破坏性删列
- 新工作台、新 API 与新计算逻辑不再依赖这些旧字段
- 月度导入模板按“当前单位下本年度有效员工 x 12 月”生成，并预填当前已有月度数据
- 月度导入缺失的员工-月份行按显式 0 值记录写入，不按忽略处理
- `supplementary_salary_income` 兼容映射到 `otherIncome`
- `supplementary_remark` 兼容映射到 `otherIncomeRemark`
- `supplementary_withheld_tax_adjustment` 与 `supplementary_source_period_label` 在新主流程中视为废弃字段
- 单位备份只导出目标单位及其关联税率版本 / 审计日志，不导出整库 `app_preferences` 与其他单位数据
- 年度工作台保存接口与单月保存接口都会阻断未确认的“入职前 / 离职后收入月份”写入
- UI 布局偏好统一写入本地 `app_preferences`，键名使用 `ui_sidebar_collapsed`、`ui_nav_order`、`ui_layout::<scope>`、`ui_dialog::<scope>`

## 6. 关键非功能要求

- 全离线运行
- Windows 优先
- 桌面 UI 适配常见 PC 分辨率
- 所有关键路径可测试、可回退
- 宽工作区支持自由布局；窄窗口退回单列堆叠并暂停自由拖拽
- 单位备份 ZIP 在 Windows 环境下通过 PowerShell 原生命令生成
- 确认后的月份数据不可被普通录入流程修改
- 当前年度结果快照必须同时匹配当前税率签名和当前数据签名，才能视为可确认结果
