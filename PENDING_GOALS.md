## 1. 未落地的核心模块

### 1.1 批量导入模块
- 当前状态：
  - 前端只有占位入口
  - 后端没有导入相关路由
  - 数据库没有导入暂存表
- 具体未完成项：
  - 导入模板下载
  - 员工基础信息批量导入
  - 月度数据批量导入
  - 导入文件解析与字段校验
  - 导入预览页
  - 冲突识别规则
    - 工号冲突
    - 证件号冲突
    - 同员工同年月记录覆盖冲突
  - 冲突处理策略
    - 跳过
    - 覆盖
    - 终止导入
  - 导入结果回执
    - 成功条数
    - 失败条数
    - 失败原因列表
- 完成标准：
  - 用户能下载模板、上传文件、看到预览、处理冲突并落库

### 1.3 系统维护模块
- 当前状态：
  - 前端已提供可编辑页面，可查看、编辑并保存全局税标
  - 前端已支持维护全局说明文本
  - 后端已提供 `GET /api/tax-policy` 和 `PUT /api/tax-policy`
  - 当前税标使用 `app_preferences` 中的 `tax_policy_settings` 键持久化
  - 当前全局说明使用 `app_preferences` 中的 `tax_policy_maintenance_notes` 键持久化
  - 保存税标后，会通过 `policy_signature` 逻辑失效旧税标结果，而不是直接删除
- 具体未完成项：
  - 税标版本管理策略
  - 更细粒度的结果失效 / 重算策略（按单位 / 年度）
  - 更友好的税标编辑体验与字段校验提示
  - 富文本说明维护
- 完成标准：
  - 用户能在系统维护页修改全局税标和说明，且计算中心与首页同步使用新税标

### 1.4 快速计算模块
- 当前状态：
  - 首页只有占位说明
  - 没有单独页面
  - 没有临时计算模型
- 具体未完成项：
  - 快速计算页
  - 一次性输入表单
  - 双方案比较结果
  - 不落库的临时结果展示
  - 清空与重算交互
- 完成标准：
  - 用户能在不写入数据库的前提下完成即时测算

## 4. 建议优先顺序

1. 系统维护
- 理由：
  - 历史查询最小闭环已落地
  - 当前剩余未落地核心模块里，系统维护是影响面最大、优先级最高的一项

2. 批量导入
- 理由：
  - 能显著降低录入成本
  - 对财务场景实际价值高

3. 导出与结果体验继续优化
- 理由：
  - 当前已可用，但仍有模板、性能和交付体验优化空间
